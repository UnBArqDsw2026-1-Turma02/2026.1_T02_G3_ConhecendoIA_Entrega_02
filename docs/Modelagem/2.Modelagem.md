# 2. Desenho de Software (Modelagem)

## Introdução
O Módulo de Desenho de Software representa a etapa de transposição dos requisitos de negócio para especificações técnicas detalhadas. Esta fase é crucial para o projeto **ConhecendoIA**, pois define como os componentes se estruturam, como os dados transitam e como os processos operacionais são executados, garantindo uma base sólida para a implementação futura.

## Metodologia
A equipe utilizou a notação **UML 2.5** como padrão para todas as representações visuais, garantindo uma linguagem universal e técnica. Foram empregadas ferramentas de alta fidelidade como **Lucidchart** e **Draw.io**, permitindo a criação de diagramas que facilitam o entendimento tanto da equipe de desenvolvimento quanto dos demais stakeholders do projeto. O processo seguiu uma abordagem iterativa, com revisões constantes para manter o alinhamento com os artefatos de elicitação.

## Rastreabilidade e Senso Crítico
Cada modelo apresentado foi construído mantendo um elo claro com artefatos anteriores, como o **Rich Picture** e as **Histórias de Usuário**. Nossa análise crítica focou em simplificar a arquitetura para evitar o *over-engineering*, priorizando soluções que garantam a escalabilidade da plataforma sem comprometer a clareza do código e a facilidade de manutenção.
